# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- `install-skill` CLI subcommand: copies the bundled `SKILL.md` to
  `~/.claude/skills/skill-miner/` so Claude Code can auto-invoke
  skill-miner when the user asks "what am I doing repeatedly?".
- Bundled `src/skill_miner/resources/skill-miner/SKILL.md` template
  (shipped via `package-data`).
- Probe-phrase filter for exploratory prompts like "你能帮我做什么" /
  "what can you do" — these are repeated across sessions but never
  represent a real workflow.
- Context-compaction marker filter: prompts that start with "This
  session is being continued from a previous conversation" or
  "Caveat: The messages below were generated by the user while running"
  are now dropped before clustering.
- `save_state` now re-scrubs every candidate's description, example
  prompts, and rendered `skill_md` on every write. This cleans up
  legacy entries written before scrubbing was added, and entries
  that the latest scan did not touch.

### Changed
- `upsert_candidates` scrubs `example_prompts` before persisting them
  to `state.json` (previously only the rendered `SKILL.md` body was
  scrubbed, leaking PII into the state file and the markdown report).

### Fixed
- Real IP addresses, SSH passwords, and tokens could leak into
  `~/.claude/mined-skills/state.json` via the `description` and
  `example_prompts` fields. Both fields now go through `synthesizer.scrub`.

## [0.1.0] - 2026-05-19

### Added
- Initial alpha release.
- `skill-miner scan` — load `~/.claude/projects/*/*.jsonl`, mine
  recurring prompt clusters and tool n-grams, persist candidates.
- `skill-miner list` — table of current candidates.
- `skill-miner report` — markdown report under
  `~/.claude/mined-skills/report-YYYY-MM-DD.md`.
- `skill-miner accept <id>` — promote a candidate to
  `~/.claude/skills/<name>/SKILL.md`.
- `skill-miner ignore <id>` — mark a candidate so re-scans skip it.
- `skill-miner install-hook` — register a Claude Code Stop hook so
  every session ends with a quiet incremental re-scan.
- `skill-miner hook` — internal entry point invoked by the Stop hook;
  swallows all exceptions so a broken miner never blocks Claude Code.
- TF-IDF + cosine clustering for prompts (stdlib only, CJK bigrams
  handled).
- 3-to-5-gram tool-call mining with per-session deduplication.
- PII scrubbing: emails, IPs, Windows + POSIX paths, `~/...` paths,
  `password=` / `token=` k/v pairs, long alphanumeric blobs.
- Status-preserving `upsert_candidates`: re-scanning never resets a
  candidate that was previously `accepted` or `ignored`.
